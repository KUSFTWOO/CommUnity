## 구현 기능

### 1. 대시보드 메인 페이지 (/dashboard/)
- **통계 위젯**
  - 총 회원 수
  - 오늘 새 게시글 수
  - 진행 중인 투표 수
  - 진행 중인 설문 수
  - 다가오는 일정 수

- **최근 활동 피드**
  - 최근 5개 게시글 (제목, 작성자, 작성일, 조회수)
  - 최근 5개 댓글 (내용, 게시글, 작성자, 작성일)
  - 최근 5개 가입 회원 (닉네임, 이메일, 가입일, 상태)

### 2. 회원 관리 페이지 (/dashboard/users/)
- 회원 검색 (닉네임, 이메일)
- 정렬 기능 (최근 가입순, 오래된 순, 닉네임순)
- 회원 목록 테이블
  - 닉네임, 이메일, 가입일, 게시글 수, 댓글 수, 권한, 상태
- 페이징 (20명씩)
- 회원 상세 페이지 (/dashboard/users/<id>/)
  - 기본 정보 (닉네임, 이메일, 이름, 소개, 가입일, 마지막 접속)
  - 권한 및 상태 관리 (관리자 권한 부여/제거, 계정 활성화/비활성화)
  - 활동 통계 (게시글, 댓글, 투표, 설문, 좋아요)

### 3. 공지사항 관리 (/dashboard/notices/)
- 공지사항 목록
- 공지사항 상세 및 수정 (/dashboard/notices/<id>/)
  - 상단 고정 토글
  - 제목/내용 수정
  - 소프트 삭제

### 4. 게시글 관리 (/dashboard/posts/)
- 게시글 검색 (제목, 내용)
- 정렬 기능 (최신순, 오래된 순, 조회수 높은 순)
- 게시글 목록 (제목, 작성자, 작성일, 조회수, 댓글 수)
- 페이징 (20개씩)
- 게시글 상세 (/dashboard/posts/<id>/)
  - 게시글 조회 (제목, 내용, 작성자, 조회수, 좋아요)
  - 댓글 목록 및 관리 (댓글 삭제)
  - 게시글 소프트 삭제

### 5. 일정 관리 (/dashboard/calendar/)
- 일정 목록 (제목, 기간, 장소, 작성자)
- 일정 상세 및 수정 (/dashboard/calendar/<id>/)
  - 기본 정보 (제목, 기간, 장소, 작성자, 작성일)
  - 설명 수정
  - 소프트 삭제

### 6. 투표 관리 (/dashboard/polls/)
- 투표 목록 (질문, 작성자, 마감일, 투표수, 상태)
- 투표 상세 (/dashboard/polls/<id>/)
  - 투표 정보 (질문, 작성자, 마감일, 상태)
  - 투표 결과 (옵션별 선택 수, 퍼센티지, 진행률 바)
  - 투표 참여자 목록

### 7. 설문조사 관리 (/dashboard/surveys/)
- 설문 목록 (제목, 작성자, 마감일, 응답 수, 상태)
- 설문 상세 (/dashboard/surveys/<id>/)
  - 설문 정보 (제목, 설명, 작성자, 마감일, 익명/실명)
  - 설문 문항 (질문, 유형, 응답 수, 선택지)
  - 응답 현황

### 8. UI/UX
- **대시보드 기본 레이아웃**
  - 좌측 사이드바 고정 네비게이션
  - 사이드바 메뉴 (대시보드, 회원 관리, 콘텐츠, 투표/설문)
  - 메인 콘텐츠 영역 (유연한 너비)

- **스타일링**
  - Tailwind CSS 기반 카드형 위젯
  - 통계 카드 (큰 숫자, 라벨)
  - 테이블 (헤더, 행, 호버 효과)
  - 배지 (상태 표시: 활성/비활성, 관리자/일반, 완료/진행중)
  - 버튼 (기본/위험 스타일)

- **반응형 디자인**
  - 데스크톱: 250px 사이드바 + 유연한 콘텐츠
  - 태블릿: 축소된 사이드바 + 콘텐츠
  - 모바일: 사이드바 아래로, 풀 너비 콘텐츠

## 발생한 문제 및 해결

### 문제 1: NoReverseMatch 에러
**증상**: 템플릿에서 {% url %} 태그 사용 시 URL을 찾을 수 없음
**원인**: Django 개발 서버의 StatReloader가 모듈 변경을 감지하지 못함
**해결**: views.py 파일에 주석 추가하여 파일 수정 트리거, 서버 자동 재로드

### 문제 2: 외부 앱 URL 참조 오류
**증상**: notices:create, votes:create 등 다른 앱의 URL을 대시보드에서 참조하려고 함
**원인**: 대시보드는 관리 인터페이스이므로 외부 앱의 생성 페이지 직접 링크 불필요
**해결**: 대시보드 템플릿에서 외부 앱 URL 참조 제거

## 배운 점

1. **Django StatReloader**: 파일 변경을 감지하여 자동 재로드하지만, 특정 상황에는 명시적 트리거 필요
2. **Template Tag Caching**: Django 템플릿이 컴파일된 후 서버 재시작까지 캐시됨
3. **URL Namespace**: 앱별 URL namespace를 이용한 동적 URL 역참조의 중요성
4. **Admin UI 패턴**: Django Admin과 유사한 패턴으로 사용자 친화적 인터페이스 설계
