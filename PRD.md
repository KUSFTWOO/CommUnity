# PRD — Product Requirements Document
## 커뮤니티 웹사이트 (CommUnity)

> 버전: 1.0.0 | 작성일: 2025-05-15 | 상태: 초안(Draft)

---

## 1. 프로젝트 개요

### 1.1 배경 및 목적
소규모 커뮤니티를 위한 통합 온라인 공간을 구축한다.
공지사항 전달, 자유로운 소통, 일정 공유, 의견 수렴(투표/설문), 회원 관리를
하나의 플랫폼에서 제공한다.

### 1.2 핵심 가치
- **접근성** — 누구나 쉽게 사용할 수 있는 직관적인 UI
- **신뢰성** — 안정적인 운영과 보안이 보장된 환경
- **확장성** — 기능을 점진적으로 추가할 수 있는 구조
- **경량성** — 불필요한 복잡도 없이 빠르게 동작

### 1.3 대상 사용자

| 역할 | 설명 |
|------|------|
| 비회원 | 공지사항·게시글 열람만 가능 (읽기 전용) |
| 일반 회원 | 게시글 작성, 댓글, 투표 참여, 일정 확인 |
| 관리자 | 공지 작성, 회원 관리, 게시글 통제, 투표 생성 |

## 3. 비기능 요구사항

### 3.1 성능
- 페이지 최초 로딩: 3초 이내
- API 응답: 500ms 이내 (일반 CRUD)
- 이미지는 업로드 시 자동 리사이즈 (최대 1200px)

### 3.2 보안
- HTTPS 필수 (배포 시 SSL 인증서)
- CSRF 토큰 모든 폼에 적용
- SQL Injection: ORM으로 방어
- XSS: Django 템플릿 자동 이스케이프
- 파일 업로드: 확장자 + MIME 타입 이중 검증

### 3.3 접근성
- 모바일 반응형 (브레이크포인트: 375px / 768px / 1280px)
- 최소 WCAG 2.1 AA 수준 색상 대비

### 3.4 확장성
- SQLite → PostgreSQL 전환 시 코드 변경 없음 (ORM 추상화)
- 로컬 파일 저장 → AWS S3 전환 가능 구조
- Django App 단위 분리로 독립적 기능 추가 가능
