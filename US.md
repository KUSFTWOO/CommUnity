# US — User Stories
## 사용자 관점의 기능 요구사항과 인수 조건

> 버전: 1.0.0
> 형식: "나는 [역할]로서, [목적]을 위해 [기능]을 원한다."

---

## 등장인물 정의

| 역할 | 설명 |
|------|------|
| **비회원** | 로그인하지 않은 방문자 |
| **회원** | 가입 후 로그인한 일반 사용자 |
| **관리자** | is_staff=True인 운영자 |

## Epic 1: 회원 시스템

### US-A01 회원가입
**나는 비회원으로서, 커뮤니티에 참여하기 위해 회원가입을 하고 싶다.**

```
Given  회원가입 페이지에 접근했을 때
When   이메일, 닉네임, 비밀번호, 비밀번호 확인을 입력하고 제출하면
Then   계정이 생성되고 자동으로 로그인되며 메인 페이지로 이동한다

Given  이미 사용 중인 이메일을 입력했을 때
Then   "이미 사용 중인 이메일입니다" 오류 메시지가 표시된다

Given  비밀번호와 비밀번호 확인이 다를 때
Then   "비밀번호가 일치하지 않습니다" 오류 메시지가 표시된다

Given  비밀번호가 8자 미만일 때
Then   "비밀번호는 8자 이상이어야 합니다" 오류 메시지가 표시된다
```

---

### US-A02 로그인
**나는 비회원으로서, 내 계정에 접근하기 위해 로그인을 하고 싶다.**

```
Given  올바른 이메일과 비밀번호를 입력하면
Then   이전 페이지 또는 메인 페이지로 이동하며 상단에 닉네임이 표시된다

Given  잘못된 비밀번호를 5회 입력했을 때
Then   계정이 30분 잠기고 "계정이 잠겼습니다. 30분 후 다시 시도해주세요" 메시지가 표시된다
```

---

### US-A03 로그아웃
**나는 회원으로서, 보안을 위해 로그아웃을 하고 싶다.**

```
Given  로그인한 상태에서 로그아웃 버튼을 클릭하면
Then   세션이 종료되고 메인 페이지로 이동하며 로그인 상태가 해제된다
```

---

### US-A04 프로필 관리
**나는 회원으로서, 나를 표현하기 위해 프로필을 수정하고 싶다.**

```
Given  내 프로필 페이지에서 닉네임, 한 줄 소개, 프로필 이미지를 변경하고 저장하면
Then   변경 사항이 즉시 반영되고 "프로필이 업데이트되었습니다" 메시지가 표시된다

Given  허용되지 않는 형식의 이미지를 업로드하면
Then   "JPG, PNG, GIF, WEBP 형식만 허용됩니다" 오류가 표시된다
```

## Epic 2: 공지사항

### US-N01 공지사항 열람
**나는 비회원/회원으로서, 중요한 소식을 확인하기 위해 공지사항을 보고 싶다.**

```
Given  공지사항 목록 페이지에 접근했을 때
Then   공지가 최신순으로 나열되고, 핀 고정 공지는 항상 상단에 표시된다

Given  공지사항 제목을 클릭하면
Then   상세 페이지에서 제목, 내용, 작성일, 첨부파일을 확인할 수 있다
```

---

### US-N02 공지사항 작성
**나는 관리자로서, 구성원에게 정보를 전달하기 위해 공지를 작성하고 싶다.**

```
Given  관리자로 로그인한 상태에서 공지 작성 페이지에서
When   제목, 내용, (선택) 파일 첨부, (선택) 핀 고정 설정 후 등록하면
Then   공지사항이 목록에 나타나고 "공지사항이 등록되었습니다" 메시지가 표시된다

Given  일반 회원이 공지 작성 URL에 직접 접근하면
Then   403 Forbidden으로 접근이 차단된다
```

## Epic 3: 자유게시판

### US-B01 게시글 목록 보기
**나는 비회원/회원으로서, 커뮤니티 소통을 확인하기 위해 게시글 목록을 보고 싶다.**

```
Given  게시판 페이지에 접근하면
Then   게시글이 최신순으로 나열되고 제목, 작성자, 작성일, 조회수, 댓글 수, 좋아요 수가 표시된다

When   "인기순" 정렬을 선택하면
Then   좋아요 수 기준으로 내림차순 정렬된다

When   검색어를 입력하면
Then   제목 또는 내용에 검색어가 포함된 게시글만 표시된다
```

---

### US-B02 게시글 작성
**나는 회원으로서, 의견을 공유하기 위해 게시글을 작성하고 싶다.**

```
Given  로그인 상태에서 글쓰기 버튼을 클릭 후
When   제목과 내용을 입력하고 (선택) 이미지 최대 5장 첨부 후 등록하면
Then   게시글이 목록에 나타나고 상세 페이지로 이동한다

Given  비로그인 상태에서 글쓰기를 시도하면
Then   로그인 페이지로 리다이렉트된다

Given  이미지를 5장 초과 첨부하면
Then   "이미지는 최대 5장까지 첨부할 수 있습니다" 오류가 표시된다
```

---

### US-B03 좋아요
**나는 회원으로서, 마음에 드는 게시글에 공감을 표현하기 위해 좋아요를 누르고 싶다.**

```
Given  게시글 상세 페이지에서 좋아요 버튼을 클릭하면
Then   좋아요 수가 1 증가하고 버튼이 활성화 상태로 변한다

Given  이미 좋아요를 누른 게시글에서 다시 클릭하면
Then   좋아요가 취소되고 수가 1 감소한다

Given  비로그인 상태에서 좋아요를 클릭하면
Then   로그인 페이지로 이동한다
```

---

### US-B04 댓글과 대댓글
**나는 회원으로서, 게시글에 의견을 남기기 위해 댓글을 작성하고 싶다.**

```
Given  댓글 입력창에 내용을 입력하고 등록하면
Then   댓글이 시간순으로 표시되고 댓글 수가 증가한다

Given  댓글의 "답글" 버튼을 클릭하면
Then   해당 댓글 하위에 대댓글 입력창이 표시된다

When   대댓글을 등록하면
Then   원 댓글 아래 들여쓰기 형태로 대댓글이 표시된다

Given  본인 댓글에서 삭제 버튼을 클릭하면
Then   확인 후 댓글이 삭제된다

Given  타인의 댓글에는
Then   수정·삭제 버튼이 표시되지 않는다
```

## Epic 4: 캘린더

### US-C01 일정 확인
**나는 비회원/회원으로서, 커뮤니티 일정을 파악하기 위해 캘린더를 보고 싶다.**

```
Given  캘린더 페이지에 접근하면
Then   현재 월의 달력이 표시되고 일정이 있는 날에 색상 표시가 된다

When   일정이 있는 날짜를 클릭하면
Then   해당 날의 일정 목록이 표시된다

When   이전/다음 달 버튼을 클릭하면
Then   해당 월의 달력으로 이동한다
```

---

### US-C02 일정 관리 (관리자)
**나는 관리자로서, 구성원의 참여를 유도하기 위해 일정을 등록하고 싶다.**

```
Given  일정 추가 폼에서 제목, 시작일, 종료일, 설명, 색상 태그를 입력하고 저장하면
Then   달력에 일정이 표시된다

When   기존 일정을 수정하면
Then   변경 사항이 달력에 즉시 반영된다
```

## Epic 5: 투표/설문

### US-V01 투표 참여
**나는 회원으로서, 커뮤니티 의사결정에 참여하기 위해 투표를 하고 싶다.**

```
Given  진행 중인 투표에 접근하면
Then   투표 항목 목록이 표시된다

When   항목을 선택하고 투표 버튼을 클릭하면
Then   "투표가 완료되었습니다" 메시지와 현재 결과(퍼센트 바)가 표시된다

Given  이미 투표한 투표에 다시 접근하면
Then   이미 선택한 항목이 표시되고 재투표할 수 없다

Given  투표 종료일이 지난 투표는
Then   투표 버튼이 비활성화되고 최종 결과만 표시된다
```

---

### US-V02 투표 생성 (관리자)
**나는 관리자로서, 구성원의 의견을 수렴하기 위해 투표를 만들고 싶다.**

```
Given  제목, 선택 항목(2개 이상), 시작일, 종료일, 익명 여부를 설정하고 등록하면
Then   투표가 진행 중 목록에 나타나고 회원들이 참여할 수 있다

When   복수 선택 허용 옵션을 켜면
Then   회원이 여러 항목을 동시에 선택할 수 있다
```

---

## Epic 6: 관리자 대시보드

### US-AD01 운영 현황 파악
**나는 관리자로서, 커뮤니티 상태를 파악하기 위해 대시보드를 보고 싶다.**

```
Given  관리자 대시보드에 접근하면
Then   오늘 신규 가입자 수, 오늘 게시글 수, 진행 중인 투표 수, 신고 대기 건수가 표시된다
```

---

### US-AD02 회원 관리
**나는 관리자로서, 문제 회원을 처리하기 위해 회원 목록을 관리하고 싶다.**

```
Given  회원 관리 페이지에서 특정 회원을 검색하면
Then   이메일, 닉네임, 가입일, 게시글 수, 현재 상태가 표시된다

When   회원 상태를 "정지"로 변경하면
Then   해당 회원은 로그인 시 "계정이 정지되었습니다" 메시지와 함께 접근이 차단된다
```

---

### US-AD03 신고 처리
**나는 관리자로서, 건전한 커뮤니티를 위해 신고된 게시글을 처리하고 싶다.**

```
Given  신고 목록에서 신고된 게시글을 확인하면
Then   신고 사유, 신고자, 신고 일시, 해당 게시글 링크가 표시된다

When   "게시글 삭제" 버튼을 클릭하면
Then   해당 게시글이 비공개 처리되고 신고 상태가 "처리 완료"로 변경된다
```
